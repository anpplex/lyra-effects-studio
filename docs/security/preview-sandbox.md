# Preview sandbox security

Theme source is untrusted even when it comes from a signed Pack. Studio renders it in an iframe with only `sandbox="allow-scripts"`; omitting `allow-same-origin` gives the document an opaque origin and prevents it from accessing Studio state, storage, cookies or the parent DOM.

The generated preview document applies a deny-by-default Content Security Policy:

- network connections, media, forms, objects and base URL changes are disabled;
- images and fonts are limited to embedded `data:` or `blob:` content;
- inline theme styles are allowed because CSS is the Pack's supported extension surface;
- scripts require a random Studio-generated nonce.

Theme HTML is inserted without that nonce, so authored script elements cannot execute. The sole privileged script is generated by Studio and installs a read-only `window.lyraBridge` with cloned scenario, now-playing and lyric data. It does not expose filesystem, Registry, signing or Tauri APIs.

The iframe reports readiness, runtime errors and policy violations through `postMessage`. Studio accepts an event only when both the sending window and the random preview token match the active iframe. The channel is diagnostics-only and cannot request privileged operations.

Scenario JSON is parsed and validated in Rust when a project opens and again before save. During editing, the frontend uses a conservative shape check; malformed drafts remain visible in source diagnostics while the preview keeps the last valid scenario.
